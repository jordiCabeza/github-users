About

GitHub User Search consist in an .NET MVC5 application that loads a SPA that communicates with Web API 2 services

Backend

MVC5

The MVC5 applications contains the HomeController with and index action to load the html needed to run the SPA.

We are not using Nuget to handle any frontend dependencies and for simplicity there is no unit test for the HomeController.

We are also using default RouteConfig.

Web API 2

For the Web API 2 services we are using:

  • StructureMap as IoC/DI
  • UnitTests project using NUnit as a unit testing framework and Moq as a mocking framework

Frontend

The frontend consist in a SPA build using the following technologies:
  • Using npm and jspm as a packet managers. Npm to handle dev dependencies like gulp(javascript task runner), karma(test runner) and jspm (frontend package manager). Jspm is use to install the needed packages (Knockout, Bootstrap, etc,..) and create the config for the SystemJS module loader. Both npm and jspm packages are not included in the zip.
  • Use SystemJS module loader to load ES2015 modules and use babel to transpile the code to ES5 for compatibility with all browsers.
  • Knockout as a MVVM framework to connect the UI with the data models. We are using knockout components to have a clean and organized code separating each teamplate in an .html file and the viewModel in a .js. You can find the components inside the folder /js/components.
  • Code divided in ES2015 modules that facilitates the testing.
  • Use karma to run the units test written in jasmine. The spec files are hosted in the same folder than the source code for simplicity

How to run unit tests

Install nodejs and npm in your machine. https://docs.npmjs.com/getting-started/installing-node

Install gulp and jspm as a global package

npm install -g gulp jspm

Open a console and inside the "Calculator" project folder install npm packages

npm install

Install jspm packages

jspm install

Finally, run gulp task to run the unit tests with karma

gulp karma
